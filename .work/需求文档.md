原始需求：
    开发一款性能较好的 markdown 软件，要能有 vscode那样可以极速加载打开目录下文件信息的能力，还要能打开较大的 markdown 文件。具体要求如下：
    1.开发应用跨平台
    2.能够极速vscode那样可以极速加载打开目录下文件信息的能力，可以在树组件下流畅操作目录文件
    3.增加 git 能力。打开的目录如果没有 git，有相应的按钮可以初始化 git，从而实现 git 管理；如果有 git，则使用现有的 git 进行管理。应用可以设置 git 是否自动同步，支持自定义同步频率或是触发条件，例如保存时在提交，提交描述用提交详细时间作为提交内容。支持 git差异内容比较
    4.应用界面与 vscode 类似，左边是属性文件夹信息，右边是文件内容，支持 tab多个文件同时打开,较多文件时支持类似 vscode 快捷键快速搜索
    5.支持类似 vscode 内容过滤搜索
    6.支持多语言，初始支持中文和英文
    7.支持从剪切板快速复制文件到文件内容中，文件通过自定义规则（自定义路径及命名），自动保存到指定路径下，文件中自动生成对应的相对链接；支持自定义文件同步方式，例如 cdn 等，在文件内容中使用网络地址，文件中特别是图片文件，需要支持无损压缩
    8.支持设置多种同步方式，git/webdav/s3/ftp等
    9.除了从本地文件夹中打开外，支持从上面同步方式中打开，默认同步方式是原打开方式，支持切换，例如从 git参考打开的文档，修改后支持同步更新到其他地址例如 webdav
    10.支持工作区模式
    11.支持 ai 设置，支持本地模型和 api 模型。
    12.支持新建文件使用 ai 后通过 url快速获取页面内容并总结生成对应的 markdown 内容，并写入文件
    13.应用要求技术栈：tauri 2.x + React 18 + Vite + TypeScript + Tailwind CSS v3 + Zustand + BlockNote + Lucide + shadcn/ui+codemirror6.0
    14.资源管理节点上添加右键能力，支持：打开、在文件资源管理器中显示、打开历史记录、复制文件、剪切文件、复制路径、复制相对路径、重命名、删除


右键菜单使用shadcn下contextMenu 实现并在复制菜单前添加粘贴菜单，点击粘贴菜单时可以将复制文件粘贴到目录下

1.文件树组件原选中逻辑没有了
2.文件重命名后自动聚焦到输入框并选中文件名，失去焦点自动确认
3.如果文件 在tab中已经打开，重命名后应该要及时响应 

文件数点击节点应该有选中状态，并且如果是文件夹应该要展开/关闭
将 侧边栏ai 图标移动至应用标题栏主题按钮前面，并点击 ai 图标按钮时在编辑区右边新增一个宽 300px 的交互区

1.源代码管理下新增git提交信息输入框和同步按钮，参考vscode
2.扫描打开文件夹下所有git仓库(这里只显示仓库名，git地址及提交状态（仓库名和git地址是上下布局），不需要提交不显示状态，待提交显示红色圆点，已提交显示绿色圆点，鼠标移动到仓库名时popover显示对于仓库目录及待提交文件数量)，并列举显示在源代码管理下


在资源管理器下文件右键上新增“同步初始化”菜单，点击后在选择文件夹下初始化git仓库，这个只有文件夹节点可用
