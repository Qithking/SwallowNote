3. 界面样式规范（适配 shadcn/ui）
   
   SwallowNote 界面样式适配 shadcn/ui 组件库，遵循其设计理念和组件规范。
   
   3.1 设计理念
   - 设计系统基石：提供基础组件层，可自定义、扩展和构建
   - 开源开放：组件源码完全开放，可直接复制使用
   - 起点而非终点：基于 shadcn/ui 打造自己的风格
   
   3.2 核心组件
   | 组件类型 | 组件名称 | 用途 |
   |----------|----------|------|
   | 表单类 | Button, Input, Select, Checkbox, Radio, Toggle | 用户输入和操作 |
   | 交互类 | Dialog, Dropdown Menu, Tabs, Toggle Menu | 页面交互和导航 |
   | 功能类 | Command, Combobox, Sheet, Avatar, Badge | 特定功能展示 |
   | 数据展示 | Table, Card, Progress, Skeleton | 数据可视化 |
   
   3.3 样式定制
   - 基于 Tailwind CSS v3 构建，所有组件样式可通过 Tailwind 类名覆盖
   - 使用 CSS 变量实现主题定制，支持暗色模式
   - 组件采用 Radix UI 原语，确保无障碍访问支持
   - 按需引入，零不必要的依赖
   
   3.4 设计原则
   - 清晰的视觉层次和分组
   - 明确的输入提示和状态反馈
   - 一致的间距和排版
   - 重视安全相关视觉提示
