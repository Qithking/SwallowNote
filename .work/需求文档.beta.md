# SwallowNote 需求规格说明书

## 1. 项目概述

### 1.1 项目背景

SwallowNote 是一款高性能的跨平台 Markdown 编辑器，旨在提供类似 VSCode 的极速文件浏览体验和强大的 Git 集成能力，专注于为开发者和写作者提供流畅的 Markdown 编辑体验。

### 1.2 产品定位

- **目标用户**：开发者、技术写作者、笔记爱好者
- **核心价值**：极速文件加载、大文件支持、Git 无缝集成、类 VSCode 操作体验
- **差异化优势**：专注 Markdown 场景的性能优化、内置 Git 工作流、多源同步能力、AI 智能辅助

## 3. 技术需求

### 3.1 跨平台支持

- [ ] Windows 10/11
- [ ] macOS 10.15+
- [ ] Linux (Ubuntu 20.04+, CentOS 7+)
- [ ] 统一安装包格式（.exe/.dmg/.deb/.rpm）

### 3.2 性能指标

- [ ] 应用启动时间 < 2s（冷启动）
- [ ] 文件树加载 1000 个文件 < 500ms
- [ ] 大文件（10MB）打开时间 < 3s
- [ ] 编辑操作响应延迟 < 50ms
- [ ] 内存占用 < 500MB（常规使用场景）
- [ ] CPU 占用率 < 30%（空闲状态）
- [ ] 同步操作不阻塞主线程
- [ ] AI 请求超时时间 < 30s（可配置）

### 3.3 技术栈选型参考

根据设计文档，以下技术栈可供评估：

- **Markdown 解析**：markdown-it / Milkdown / Cherry Markdown
- **富文本编辑**：Milkdown（基于 ProseMirror）
- **桌面框架**：Electron / Tauri（需评估性能）
- **前端框架**：React / Vue（需评估生态）
- **Git 操作**：isomorphic-git / simple-git
- **文件监控**：chokidar
- **WebDAV 客户端**：webdav-client
- **S3 客户端**：aws-sdk / @aws-sdk/client-s3
- **FTP 客户端**：basic-ftp / ssh2-sftp-client
- **国际化**：i18next / vue-i18n
- **图片处理**：sharp / image-size
- **AI 集成**：
  - OpenAI SDK
  - LangChain.js
  - Ollama API 客户端
  - Puppeteer / Playwright（网页抓取）
  - Cheerio（HTML 解析）

### 3.4 数据安全

- [ ] 本地数据存储，不上传云端（除非用户主动配置同步）
- [ ] 异常退出时自动恢复未保存内容
- [ ] 定期备份机制（可选）
- [ ] 敏感信息加密存储（Git Token、WebDAV 密码、S3 Key、API Key 等）
- [ ] 同步传输使用 HTTPS/SSH 加密
- [ ] 同步日志脱敏处理
- [ ] AI API Key 加密存储，不在日志中明文显示
- [ ] URL 抓取遵守 robots.txt 协议

### 3.5 AI 伦理与合规

- [ ] 明确标注 AI 生成内容
- [ ] 提供 AI 使用指南和最佳实践
- [ ] 尊重版权，URL 抓取时注明来源
- [ ] 用户数据隐私保护，不用于模型训练

## 8. 附录

### 8.1 术语说明

- **虚拟滚动**：仅渲染可视区域内容的技术
- **Git 工作区**：包含已跟踪和未跟踪文件的目录
- **差异对比**：显示文件修改前后变化的视图
- **WebDAV**：基于 HTTP 的网络文件系统协议
- **S3**：Amazon Simple Storage Service，对象存储服务
- **工作区**：包含多个根目录的项目集合
- **Token**：AI 模型处理的基本单位，约等于 0.75 个英文单词
- **Temperature**：AI 生成文本的随机性参数，值越高越有创造性
