# 执行文档

## 2026-05-16: 文件夹历史记录功能

**需求描述：** 应用打开的文件夹历史近50条记录保存到sqlite中，应用初次打开默认加载上一次打开的记录，相同路径path只能有一条数据，如果已经存在则更新打开时间

**解决方案：** 使用 rusqlite 在 Tauri AppData 目录下创建 SQLite 数据库，保存文件夹打开历史。应用启动时自动加载上次打开的文件夹，打开新文件夹时自动保存记录。

**修改内容：**

1. **新增 rusqlite 依赖** `src-tauri/Cargo.toml`
   - 添加 `rusqlite = { version = "0.31", features = ["bundled"] }`

2. **新增数据库模块** `src-tauri/src/db/mod.rs`
   - `Database` 结构体封装 SQLite 连接（Mutex 保证线程安全）
   - `init_db` 函数初始化数据库并创建 folder_history 表

3. **新增文件夹历史操作** `src-tauri/src/db/folder_history.rs`
   - `save_folder` - INSERT OR REPLACE 实现 upsert，插入后清理超过50条的旧记录
   - `get_latest_folder` - 查询最新打开的文件夹路径
   - `get_folder_history` - 查询最近50条历史记录

4. **新增 Tauri 命令** `src-tauri/src/commands/folder_history.rs`
   - `save_folder_history` - 保存文件夹打开记录
   - `get_latest_folder` - 获取最新文件夹路径
   - `get_folder_history` - 获取历史记录列表

5. **修改 lib.rs** `src-tauri/src/lib.rs`
   - 在 setup 中初始化数据库并注册到 Tauri state
   - 注册三个新的 Tauri 命令

6. **修改前端调用封装** `src/lib/tauri.ts`
   - `saveFolderHistory` - 调用保存记录命令
   - `getLatestFolder` - 调用获取最新文件夹命令
   - `getFolderHistory` - 调用获取历史记录命令

7. **修改 Workspace Store** `src/stores/workspace.ts`
   - `openFolder` - 打开文件夹并自动保存历史记录
   - `loadLastFolder` - 加载上次打开的文件夹（静默失败）

8. **修改 App.tsx** `src/App.tsx`
   - 在 useEffect 中调用 loadLastFolder 实现启动自动加载

9. **修改 FileTreeView** `src/components/FileTree/FileTreeView.tsx`
   - handleOpenFolder 改用 openFolder 方法替代 setRootPath

**数据库表结构：**
```sql
CREATE TABLE IF NOT EXISTS folder_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    path TEXT UNIQUE NOT NULL,
    opened_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX IF NOT EXISTS idx_opened_at ON folder_history(opened_at DESC);
```

**数据库位置：** Tauri AppData 目录下的 `swallownote.db`

## 2026-05-11: 修复资源管理器复制文件到剪贴板功能（第二版）

**问题描述：** 之前的剪贴板实现方案（AppleScript/JXA 调用 NSPasteboard）在 macOS Sequoia 上无法正常工作，复制文件后无法在 Finder 中粘贴。

**解决方案：** 采用自管理的复制/粘贴状态，而非依赖系统剪贴板。右键复制/剪切文件后，文件路径存储在 Zustand store 中。粘贴时通过 Tauri 命令将文件复制/移动到目标目录。

**修改内容：**

1. **新增 shadcn ContextMenu 组件** `src/components/ui/context-menu.tsx`
   - 使用 `@radix-ui/react-context-menu` 实现
   - 支持标准右键菜单样式和动画

2. **UI Store 扩展** `src/stores/ui.ts`
   - 添加 `clipboardFiles: string[]` - 存储被复制/剪切的文件路径列表
   - 添加 `clipboardIsCut: boolean` - 标记是复制还是剪切模式
   - 添加 `setClipboardFiles()` - 设置剪贴板内容
   - 添加 `clearClipboard()` - 清除剪贴板

3. **重写 FileTreeContextMenu** `src/components/FileTree/FileTreeContextMenu.tsx`
   - 使用 shadcn `ContextMenu` 组件替代自定义实现
   - 新增"粘贴"菜单项（仅在目标是目录时显示，位于复制之前）
   - 粘贴逻辑：通过 Tauri `copy_file` 命令将文件复制到目标目录
   - 剪切模式下粘贴后自动删除源文件
   - 粘贴后自动刷新目标目录内容

4. **更新 FileTreeView** `src/components/FileTree/FileTreeView.tsx`
   - 移除旧的 contextMenu 状态管理
   - 导入 `TreeNodeContextMenu` 组件，包装每个树节点
   - 监听 `filetree:rename` 事件触发重命名模式

5. **组件导出** `src/components/index.ts`
   - 导出 ContextMenu 相关组件

**工作流程：**
1. 用户右键文件 → 选择"复制"
   - 文件路径存入 `clipboardFiles`，`clipboardIsCut = false`
   - 显示 Toast 反馈
2. 用户右键目录 → 选择"粘贴"
   - 调用 Tauri `copy_file` 命令将文件复制到目标目录
   - 如果是剪切模式，删除源文件
   - 刷新目标目录
   - 显示 Toast 结果
