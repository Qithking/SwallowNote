# 执行文档

## 2026-05-11: 修复资源管理器复制文件到剪贴板功能

**问题描述：** 资源管理器下右键菜单"复制"功能没有正确复制文件到系统剪贴板。

**根因分析：**
- macOS 上 Rust 后端 `copy_file_to_clipboard` 命令使用了 AppleScript 的 `set the clipboard to` 来复制文件引用
- 该 API 在现代 macOS（Sequoia 15.x+）上已被弃用且不可靠，导致静默失败
- 同时文件路径未做转义处理，特殊字符路径也会导致失败

**修复内容：**
1. **Rust 后端 (src-tauri/src/commands/file.rs):**
   - macOS: 将 AppleScript 实现替换为 JXA (JavaScript for Automation) 方案，直接调用 `NSPasteboard.writeObjects` API，更可靠
   - 路径字符串添加转义处理（反斜杠和单引号）
   - Windows/Linux: 同步添加路径转义，将 `spawn()` 改为 `output()` 确保进程执行完毕

2. **前端 UI Store (src/stores/ui.ts):**
   - 添加 `toastMessage` 状态和 `showToast` 方法，支持临时消息提示

3. **前端 App (src/App.tsx):**
   - 添加 Toast 通知 UI 组件，2 秒自动消失

4. **前端 FileTreeContextMenu (src/components/FileTree/FileTreeContextMenu.tsx):**
   - 复制/剪切操作成功后显示 Toast 反馈

**影响范围：**
- 文件树右键菜单 → 复制（`Ctrl+C` 功能）
- 文件树右键菜单 → 剪切
- 文件树右键菜单 → 复制路径 / 复制相对路径
